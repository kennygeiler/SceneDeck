# Architecture Handoff

## Summary

SceneDeck: a searchable database of 50-100 iconic cinema shots tagged with structured camera motion metadata. Next.js 15 monolith on Vercel with offline Python data pipeline and Gemini 2.0 Flash for AI classification. Hero feature is an HTML5 Canvas + SVG layered video metadata overlay.

## Milestone Count: 7

## Milestones

1. **Foundation & Design System** -- Next.js 15 scaffold, taxonomy constants (TS + Python), design tokens, cinematic dark theme, Vercel deployment
2. **Hero Overlay** -- HTML5 Canvas + SVG video metadata overlay with motion arrows, trajectory paths, speed indicators (mock data)
3. **Database, Browse & Search** -- Neon PostgreSQL + pgvector, Drizzle ORM, browse with faceted filters, semantic NL search, landing page
4. **QA Verification** -- Human rating workflow (0-5 scale), per-field accuracy, correction submission, no auth
5. **Pipeline** -- PySceneDetect shot detection, Gemini 2.0 Flash classification, Vercel Blob upload, accuracy validation checkpoint
6. **Export** -- JSON + CSV export with filter pass-through
7. **Polish & Deploy** -- 50-100 seed shots, performance tuning, production deployment, demo script

## Execution Order

M1 -> M2 -> M3 -> M4 + M5 (parallel) -> M6 -> M7

## Tech Stack

- **Frontend**: Next.js 15 App Router, TypeScript, Tailwind CSS 4, shadcn/ui, Framer Motion
- **Database**: Neon PostgreSQL with pgvector extension, Drizzle ORM
- **Storage**: Vercel Blob (video clips + thumbnails)
- **AI Classification**: Gemini 2.0 Flash (primary), RAFT on Modal (fallback only if Gemini fails QA)
- **Pipeline**: Python, PySceneDetect, FFmpeg
- **Hosting**: Vercel
- **Overlay**: HTML5 Canvas + SVG layered architecture (ADR-008)

## Critical Constraints for Build

- **C-01**: All code generated by AI agents
- **C-03**: Gemini-first classification -- validate accuracy on 10-20 clips before scaling; do NOT build RAFT pipeline upfront
- **C-07**: No authentication required -- public access
- **C-08**: Frontend quality over pipeline completeness -- if time compresses, UI must be amazing even with manual data
- **C-12**: Hero overlay must be screen-recordable quality (no flickering, no z-index issues)
- **C-14**: Design tokens and component style guide before building UI components
