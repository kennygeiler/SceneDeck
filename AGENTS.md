## Architecture TL;DR

SceneDeck is a searchable database of iconic cinema shots tagged with structured camera motion metadata. It is a Next.js 15 App Router monolith (TypeScript, Tailwind CSS 4, shadcn/ui) deployed on Vercel, with Neon PostgreSQL + pgvector for data and semantic search. The hero feature is an HTML5 Canvas + SVG layered video metadata overlay showing movement type, direction arrows, trajectory paths, and speed indicators synchronized to video playback via requestAnimationFrame. An offline Python pipeline uses PySceneDetect for shot detection and Gemini 2.0 Flash for camera motion classification. Video clips and thumbnails are stored on Vercel Blob.

## Conventions

- **Framework**: Next.js 15 App Router only (no Pages Router). Server Components for data fetching, Client Components only for interactivity.
- **Styling**: Tailwind CSS 4 utility classes. Design tokens defined in `tokens.css` as CSS custom properties. Dark cinematic theme.
- **Components**: shadcn/ui as base component library (source files, not node_modules). Radix UI primitives underneath.
- **Animation**: Framer Motion for UI transitions and overlay state changes.
- **Database**: Drizzle ORM for type-safe queries against Neon PostgreSQL.
- **Taxonomy**: Camera movement taxonomy (21 types, 15 directions, 7 speeds, 15 shot sizes, 15 angles, 6 durations) defined as shared constants in both TypeScript and Python. These are immutable at runtime.
- **File organization**: App Router file conventions (`page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx`). Components in `src/components/`. Lib/utils in `src/lib/`.
- **Naming**: kebab-case for files, PascalCase for components, camelCase for functions/variables.
- **No auth**: Public access, no login, no protected routes (C-07).
- **No manual code**: All application code generated by AI agents (C-01).

## Taxonomy Quick Reference

- 21 movement types: static, pan, tilt, dolly, truck, pedestal, crane, boom, zoom, dolly_zoom, handheld, steadicam, drone, aerial, arc, whip_pan, whip_tilt, rack_focus, follow, reveal, reframe
- 15 direction values
- 7 speed categories
- 15 shot sizes
- 15 camera angles (6 vertical + 5 horizontal + 4 special)
- 6 duration categories
- Compound notation: max 3 simultaneous components as ordered `{type}:{direction}` pairs
