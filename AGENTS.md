## Architecture TL;DR

SceneDeck is a searchable database of iconic cinema shots tagged with structured camera motion metadata. It is a Next.js 15 App Router monolith (TypeScript, Tailwind CSS 4, shadcn/ui, Framer Motion) deployed on Vercel, with Neon PostgreSQL + pgvector for data and semantic search. The hero feature is an SVG-based video metadata overlay showing movement type, direction arrows, trajectory paths, speed indicators, and camera angles synchronized to video playback. An offline Python pipeline (in `/pipeline`) uses PySceneDetect for shot detection and Gemini 2.0 Flash for camera motion classification. Video clips and thumbnails are stored on Vercel Blob.

## Current State

M1 (Foundation) and M2 (Hero Overlay) are complete. The app has a working landing page, browse page with movement type filters, shot detail page with full metadata overlay, and 3 mock shots. All data currently comes from `src/lib/mock/shots.ts` -- no database yet. Placeholder routes exist at `/verify` and `/export`. Drizzle ORM and Neon dependencies are not yet installed.

## Conventions

- **Framework**: Next.js 15 App Router only (no Pages Router). Server Components for data fetching, Client Components only for interactivity (marked with `"use client"`).
- **Styling**: Tailwind CSS 4 utility classes. Design tokens in `src/styles/tokens.css` as CSS custom properties (OKLCH color space). Dark cinematic theme. No pure black/white. No hardcoded colors outside token set.
- **Components**: shadcn/ui as base component library (source in `src/components/ui/`). Radix UI primitives underneath.
- **Animation**: Framer Motion for UI transitions and overlay state changes.
- **Database**: Drizzle ORM for type-safe queries against Neon PostgreSQL (to be set up in M3).
- **Taxonomy**: Camera movement taxonomy (21 types, 15 directions, 7 speeds, 15 shot sizes, 15 angles, 6 durations) defined as shared constants in both TypeScript (`src/lib/taxonomy.ts`) and Python (`pipeline/taxonomy.py`). Immutable at runtime.
- **File organization**: App Router conventions (`page.tsx`, `layout.tsx`). Components in `src/components/` by domain (layout, shots, video, ui). Lib/utils in `src/lib/`.
- **Naming**: kebab-case for files, PascalCase for components, camelCase for functions/variables.
- **Display helpers**: Use `src/lib/shot-display.ts` for taxonomy display names, duration formatting, compound notation. Do not duplicate these lookups.
- **No auth**: Public access, no login, no protected routes (C-07).
- **No manual code**: All application code generated by AI agents (C-01).

## Taxonomy Quick Reference

- 21 movement types: static, pan, tilt, dolly, truck, pedestal, crane, boom, zoom, dolly_zoom, handheld, steadicam, drone, aerial, arc, whip_pan, whip_tilt, rack_focus, follow, reveal, reframe
- 15 directions: left, right, up, down, in, out, clockwise, counter_clockwise, forward, backward, lateral_left, lateral_right, diagonal, circular, none
- 7 speeds: freeze, imperceptible, slow, moderate, fast, very_fast, snap
- 15 shot sizes: extreme_wide, wide, full, medium_wide, medium, medium_close, close, extreme_close, insert, two_shot, three_shot, group, ots, pov, reaction
- 6 vertical angles: eye_level, high_angle, low_angle, birds_eye, worms_eye, overhead
- 5 horizontal angles: frontal, profile, three_quarter, rear, ots
- 4 special angles: dutch, pov, shoulder_mounted, slanted
- 6 duration categories: flash, brief, standard, extended, long_take, oner
- Compound notation: max 3 parts as `{type}:{direction}` pairs

## Design System Contract

- **Color palette**: OKLCH tokens only. Accent cyan hue 200. Neutrals hue 260. No pure black/white.
- **Typography**: Inter Variable for headings/body. JetBrains Mono for technical data, taxonomy slugs, timecodes.
- **Spacing**: 4px base system via `--space-*` tokens.
- **Overlay colors**: `--color-overlay-arrow` (cyan), `--color-overlay-trajectory` (violet), `--color-overlay-speed` (light cyan), `--color-overlay-badge` (amber).
